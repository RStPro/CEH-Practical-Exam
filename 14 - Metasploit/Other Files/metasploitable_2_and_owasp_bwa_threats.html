Metasploitable2 & OWASP Broken Web Applications — Threats & Notes

Concise reference for what to expect inside Metasploitable2 and OWASP BWA. Both are intentionally vulnerable and contain web shells, outdated apps, and misconfigurations for training. Keep these VMs isolated.

Metasploitable2 — Quick Threat Summary

Metasploitable2 is a classic, intentionally vulnerable Linux VM (by Rapid7) designed for exploitation practice.

Common malicious artifacts & web shells

  • PHP webshells: c99.php, r57.php — common in webroot directories.
  • Backdoor scripts and reverse shells located in web folders or tmp directories.
  • Prebuilt exploit binaries and payloads left for detection practice.

Vulnerable apps & services

  • VSFTPD backdoor/backdoor-enabled version (historical)
  • Unpatched Apache 2.x with vulnerable modules
  • Old MySQL, Postgres with weak/no passwords
  • Tomcat with manager app and default creds
  • Unsecured Samba/SMB shares
  • Open services: FTP, Telnet, SSH (weak creds), SNMP, distcc, rpcbind

OWASP BWA (Broken Web Applications) — Quick Threat Summary

OWASP BWA is a collection of many intentionally insecure web applications useful for learning web vulnerabilities (XSS, SQLi, RCE, file upload, auth bypass, etc.).

Included vulnerable apps (examples)

  • DVWA (Damn Vulnerable Web App)
  • Mutillidae
  • WebGoat
  • Old Joomla, Drupal, and WordPress test instances
  • Custom vulnerable scripts demonstrating file upload, RCE, LFI/RFI

Common malicious artifacts & behaviors

  • Uploaded web shells via vulnerable file upload forms (PHP/ASP/JSP shells).
  • Deliberate SQL injection payloads in stored files or sample data.
  • Example exploit chains and scripts present as learning artifacts.

Typical scanner/AV detections you may see

  • Webshell signatures (Backdoor:ASP/Dirtelti, Backdoor.PHP, Webshell.JSP)
  • Exploit payloads flagged as Trojans or PUA (potentially unwanted)
  • Old binary libraries/drivers flagged for CVE-based heuristics

Recommended scans & quick checks

# Nmap full TCP scan and service detection
nmap -sS -p- -sV -oN metasploitable2_services.txt 192.168.56.20

# HTTP directory brute (example)
ffuf -u http://192.168.56.40/FUZZ -w /usr/share/wordlists/dirbuster/directory-list-2.3-medium.txt

# Simple grep for webshell patterns on the target (if you have console access)
grep -R --include="*.php" -n "eval|base64_decode|shell_exec|passthru" /var/www

Safety & operational advice

  • Isolation: Always run Metasploitable2 and OWASP BWA on Host-Only or Internal networks.
  • Snapshots: After a fresh deploy, take a snapshot to easily revert to a clean state.
  • No host shares: Avoid shared folders or clipboard sharing between host and these VMs.
  • Data exfiltration caution: Do not upload suspicious files from VMs to cloud or email them.
